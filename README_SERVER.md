# Инструкция по запуску аудио плеера на сервере

## Требования
- Веб-сервер (Apache, Nginx, Node.js или любой другой)
- Поддержка статических файлов (HTML, CSS, JS)

---

## Вариант 1: Запуск с помощью Python (быстрый способ для тестирования)

Если у вас установлен Python 3:

```bash
# Перейдите в директорию с файлами плеера
cd /workspace

# Запустите встроенный HTTP-сервер Python
python3 -m http.server 8000
```

Откройте браузер и перейдите по адресу: **http://localhost:8000**

---

## Вариант 2: Запуск с помощью Node.js (http-server)

```bash
# Установите http-server глобально (если ещё не установлен)
npm install -g http-server

# Перейдите в директорию с файлами плеера
cd /workspace

# Запустите сервер
http-server -p 8000
```

Откройте браузер и перейдите по адресу: **http://localhost:8000**

---

## Вариант 3: Развёртывание на Apache

1. Скопируйте файлы плеера в корневую директорию веб-сервера:
   ```bash
   cp index.html styles.css script.js /var/www/html/
   ```

2. Убедитесь, что Apache запущен:
   ```bash
   sudo systemctl start apache2
   ```

3. Откройте браузер и перейдите по адресу: **http://ваш-сервер/**

---

## Вариант 4: Развёртывание на Nginx

1. Создайте директорию для сайта:
   ```bash
   sudo mkdir -p /var/www/audioplayer
   ```

2. Скопируйте файлы:
   ```bash
   cp index.html styles.css script.js /var/www/audioplayer/
   ```

3. Создайте конфигурационный файл Nginx:
   ```bash
   sudo nano /etc/nginx/sites-available/audioplayer
   ```

4. Добавьте следующую конфигурацию:
   ```nginx
   server {
       listen 80;
       server_name ваш-домен.com;
       
       root /var/www/audioplayer;
       index index.html;
       
       location / {
           try_files $uri $uri/ =404;
       }
   }
   ```

5. Включите сайт и перезапустите Nginx:
   ```bash
   sudo ln -s /etc/nginx/sites-available/audioplayer /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

## Загрузка музыки в плеер

### Способ 1: Через интерфейс браузера (локальные файлы)

1. Откройте плеер в браузере
2. Нажмите кнопку **"📁 Добавить файлы"**
3. Выберите аудиофайлы на вашем компьютере (MP3, WAV, OGG и др.)
4. Файлы добавятся в плейлист и будут готовы к воспроизведению

> ⚠️ **Важно:** При использовании локальных файлов они загружаются через `URL.createObjectURL()` и доступны только в текущей сессии браузера. После обновления страницы файлы нужно будет добавить заново (метаданные плейлиста сохраняются).

### Способ 2: Размещение файлов на сервере

Для постоянного доступа к музыке разместите аудиофайлы на сервере:

1. Создайте директорию для музыки:
   ```bash
   sudo mkdir -p /var/www/audioplayer/music
   ```

2. Загрузите ваши аудиофайлы:
   ```bash
   cp /путь/к/вашим/файлам/*.mp3 /var/www/audioplayer/music/
   ```

3. Установите правильные права доступа:
   ```bash
   sudo chown -R www-data:www-data /var/www/audioplayer/music
   sudo chmod -R 755 /var/www/audioplayer/music
   ```

4. Теперь файлы доступны по URL: `http://ваш-сервер/music/название-файла.mp3`

### Способ 3: Модификация плеера для загрузки с сервера

Для автоматической загрузки файлов с сервера можно модифицировать `script.js`:

```javascript
// Пример функции для загрузки списка файлов с сервера
async function loadTracksFromServer() {
    const response = await fetch('/api/tracks'); // Ваш API endpoint
    const tracks = await response.json();
    
    tracks.forEach(track => {
        state.playlist.push({
            id: track.id,
            title: track.title,
            artist: track.artist,
            src: track.url, // URL на сервере
            fileName: track.fileName,
            duration: track.duration
        });
    });
    
    updatePlaylistDisplay();
}
```

## Экспорт и импорт плейлиста

### Экспорт плейлиста:
1. Нажмите кнопку **"💾 Экспорт"**
2. Браузер скачает JSON-файл со списком треков
3. Сохраните файл в надёжном месте

### Импорт плейлиста:
1. Нажмите кнопку **"📂 Импорт"**
2. Выберите ранее экспортированный JSON-файл
3. Метаданные треков будут добавлены в плейлист

> ⚠️ **Примечание:** При импорте сохраняются только метаданные (названия, исполнители). Аудиофайлы необходимо добавить повторно через "Добавить файлы".

## Особенности работы

1. **Сохранение настроек:** Громкость, режимы повтора и перемешивания сохраняются в `localStorage` браузера
2. **Метаданные плейлиста:** Сохраняются между сессиями, но сами аудиофайлы нужно загружать повторно
3. **Форматы:** Поддерживаются все форматы, которые может воспроизвести ваш браузер (MP3, WAV, OGG, AAC и др.)
4. **Кроссбраузерность:** Работает во всех современных браузерах (Chrome, Firefox, Safari, Edge)

## Устранение неполадок

### Файлы не загружаются
- Проверьте консоль браузера (F12) на наличие ошибок
- Убедитесь, что файлы имеют аудиоформат
- Проверьте права доступа к файлам на сервере

### Плеер не отображается
- Убедитесь, что все три файла (index.html, styles.css, script.js) находятся в одной директории
- Проверьте, что сервер правильно отдаёт файлы (статус 200)
- Очистите кэш браузера

### Звук не работает
- Проверьте, не выключен ли звук в системе
- Убедитесь, что громкость в плеере не на нуле
- Проверьте настройки звука браузера
