## 🎵 Функциональные возможности

### Основные функции плеера
| Функция | Описание | Реализация |
|---------|----------|------------|
| **Воспроизведение** | Play/Pause, переключение треков | HTML5 Audio API |
| **Прогресс-бар** | Отображение и перемотка | Input range + timeupdate event |
| **Громкость** | Регулировка 0-100%, mute/unmute | Volume bar + состояние |
| **Плейлист** | Добавление, удаление, порядок | Массив объектов в state |
| **Shuffle** | Перемешивание треков | Алгоритм Fisher-Yates |
| **Repeat** | 3 режима: off/all/one | Циклическое переключение |
| **Горячие клавиши** | Пробел, стрелки, M | KeyboardEvent listener |

### Функции работы с MinIO
| Функция | Endpoint | Метод | Описание |
|---------|----------|-------|----------|
| **Загрузка в облако** | `/api/tracks` | POST | Загрузка файлов в MinIO |
| **Список треков** | `/api/tracks` | GET | Получение списка из хранилища |
| **Потоковое воспроизведение** | `/api/tracks/<filename>` | GET | Stream аудио из MinIO |
| **Удаление трека** | `/api/tracks/<filename>` | DELETE | Удаление из хранилища |
| **Массовое удаление** | `/api/tracks/batch-delete` | POST | Удаление нескольких файлов |
| **Health check** | `/api/health` | GET | Проверка подключения к MinIO |

### Дополнительные возможности
- **Экспорт/Импорт плейлиста** - JSON файлы
- **Автосохранение** - LocalStorage браузера
- **Уведомления об ошибках** - Toast-сообщения
- **Очистка blob URL** - предотвращение утечек памяти
- **Метаданные из имён файлов** - парсинг "Artist - Title"

## 🔍 Детальный анализ компонентов

### 1. Frontend (script.js)

#### Состояние приложения
```javascript
const state = {
    playlist: [],              // Массив треков
    currentTrackIndex: -1,     // Индекс текущего трека
    isPlaying: false,          // Статус воспроизведения
    isShuffle: false,          // Режим перемешивания
    repeatMode: 'off',         // Режим повтора
    shuffledIndices: [],       // Перемешанный порядок
    volume: 75,                // Громкость (0-100)
    isMuted: false             // Статус mute
};
```

#### Ключевые функции
| Функция | Назначение | Особенности |
|---------|------------|-------------|
| `uploadToMinio()` | Загрузка файлов | FormData, async/await |
| `loadFromMinio()` | Получение списка | Проверка дубликатов |
| `deleteTrackFromMinio()` | Удаление трека | Вызов при удалении из UI |
| `handleFileUpload()` | Локальная загрузка | Извлечение duration |
| `playTrack()` | Воспроизведение | Проверка blob URL |
| `handleAudioError()` | Обработка ошибок | Автопереход к следующему |

#### Исправленные проблемы
1. ✅ **Утечка памяти** - `URL.revokeObjectURL()` при удалении
2. ✅ **Логика shuffle** - проверка `currentIndex === -1`
3. ✅ **Магические числа** - вынесены в константы
4. ✅ **Валидация источников** - проверка перед воспроизведением
5. ✅ **Toast уведомления** - вместо `alert()`

### 2. Backend (app.py)

#### Конфигурация MinIO
```python
MINIO_ENDPOINT = os.getenv('MINIO_ENDPOINT', 'localhost:9000')
MINIO_ACCESS_KEY = os.getenv('MINIO_ACCESS_KEY', 'minioadmin')
MINIO_SECRET_KEY = os.getenv('MINIO_SECRET_KEY', 'minioadmin')
MINIO_BUCKET = os.getenv('MINIO_BUCKET', 'music')
MINIO_SECURE = os.getenv('MINIO_SECURE', 'false').lower() == 'true'
```

#### REST API структура
| Endpoint | Метод | Статусы | Описание |
|----------|-------|---------|----------|
| `/api/health` | GET | 200 | Проверка соединения |
| `/api/tracks` | GET | 200 | Список треков |
| `/api/tracks` | POST | 201 | Загрузка файлов |
| `/api/tracks/<path>` | GET | 200/404 | Stream файла |
| `/api/tracks/<path>` | DELETE | 200/404 | Удаление |
| `/api/tracks/batch-delete` | POST | 200 | Массовое удаление |
| `/api/config` | GET | 200 | Конфигурация |

#### Особенности реализации
- **Ленивая инициализация** клиента MinIO
- **Автоматическое создание бакета** при старте
- **Потоковая передача** для больших файлов
- **Извлечение метаданных** из имени файла
- **Безопасные имена** через `secure_filename()`
- **Поддержка 8 форматов**: mp3, wav, ogg, flac, aac, m4a, wma, opus

## 🐳 Docker конфигурация

### Сервисы docker-compose.yml

#### MinIO Server
```yaml
image: minio/minio:latest
ports: 
  - "9000:9000"  # API
  - "9001:9001"  # Console
volumes: minio_data:/data
healthcheck: curl http://localhost:9000/minio/health/live
```

#### Flask App
```yaml
build: .
ports: "5000:5000"
environment:
  MINIO_ENDPOINT: minio:9000  # internal Docker network
depends_on: minio (healthy)
volumes: .:/app  # hot reload
```

### Переменные окружения
| Переменная | Default | Описание |
|------------|---------|----------|
| `MINIO_ENDPOINT` | localhost:9000 | Адрес MinIO |
| `MINIO_ACCESS_KEY` | minioadmin | Логин |
| `MINIO_SECRET_KEY` | minioadmin | Пароль |
| `MINIO_BUCKET` | music | Имя бакета |
| `MINIO_SECURE` | false | HTTPS режим |

## 🔐 Безопасность

### Текущее состояние
- ⚠️ **Default credentials** - minioadmin/minioadmin
- ⚠️ **Нет аутентификации** пользователей
- ⚠️ **Нет rate limiting** для API
- ✅ **secure_filename()** - защита от path traversal
- ✅ **Проверка расширений** - только аудиофайлы
- ✅ **CORS** - настроен явно

### Рекомендации для production
1. Сменить учётные данные MinIO
2. Добавить JWT аутентификацию
3. Внедрить rate limiting (Flask-Limiter)
4. Использовать HTTPS (MINIO_SECURE=true)
5. Валидировать размер файлов (сейчас 100MB лимит)
6. Логирование всех операций
7. Backup стратегия для бакета

## 🚀 Сценарии использования

### Сценарий 1: Локальное прослушивание
1. Открыть `index.html` в браузере
2. Нажать "📁 Добавить файлы"
3. Выбрать аудиофайлы с компьютера
4. Воспроизводить без сервера

### Сценарий 2: Облачное хранилище
1. Запустить `docker-compose up -d`
2. Открыть `http://localhost:5000`
3. Выбрать файлы → нажать "☁️ В облако"
4. Файлы сохраняются в MinIO навсегда
5. После перезагрузки: "⬇️ Из облака" → восстановить плейлист

### Сценарий 3: Совместный доступ
1. Развернуть на сервере с публичным IP
2. Настроить обратный прокси (nginx)
3. Пользователи загружают треки в общее облако
4. Все видят один плейлист
